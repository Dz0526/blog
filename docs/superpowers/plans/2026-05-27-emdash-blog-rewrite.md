# EmDash Blog Rewrite Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rewrite the dz99.me blog from Next.js 13/Vercel to EmDash CMS (Astro on Cloudflare Workers) — combining a minimal profile page with a new blog, archiving all existing content at `/archive/*`, and enabling MCP-driven authoring.

**Architecture:** Astro app integrated with EmDash CMS, deployed to Cloudflare Workers, storing content in D1 (SQLite) and media in R2. EmDash provides admin UI at `/_emdash/admin`, a built-in MCP server for agent-driven authoring, and runtime content-type definitions managed through the admin UI (with `npx emdash types` generating TypeScript types for the Astro pages to consume).

**Tech Stack:**
- **Framework:** Astro (latest stable as of cutover)
- **CMS:** EmDash (`@emdash-cms/*`, pinned to the latest stable release as of cutover; current as of plan writing: `@emdash-cms/blocks@0.14.0`, released 2026-05-20)
- **Runtime:** Cloudflare Workers
- **Storage:** Cloudflare D1 (content), R2 (media), KV (optional cache)
- **DNS:** Cloudflare (NS-only migration from お名前.com)
- **OG images:** `workers-og`
- **Testing:** Vitest (unit), Playwright (E2E)
- **Package manager:** pnpm (replaces yarn; EmDash scaffold default)

**Linked spec:** [`docs/superpowers/specs/2026-05-27-emdash-blog-rewrite-design.md`](../specs/2026-05-27-emdash-blog-rewrite-design.md)

---

## Spec deviations recorded during planning

Compared to the design spec, the following items are adjusted based on EmDash documentation research:

- **Admin URL:** `/_emdash/admin` (not `/admin`). Spec §6 updated implicitly.
- **Content type definition:** runtime / database-defined via admin UI + EmDash CLI (`npx emdash types`), not TypeScript schema files. Spec §5 still describes the **logical** model — the plan executes it through EmDash's actual mechanism.
- **EmDash version pin:** plan targets latest stable as of cutover (currently `0.14.x` line), not literal `v0.1.0`. The product matured past v0.1.0 since spec authoring.
- **Authentication:** EmDash now provides passkey/WebAuthn + OAuth + magic link out of the box. Cloudflare Access in front of `/_emdash/admin` and `/mcp/*` is still kept as defense in depth per spec §C10.

export interface ParsedDoc {
  title: string;
  date: string;
  body: string;
}

export function parseFrontmatter(input: string): ParsedDoc {
  const { data, content } = matter(input);
  if (typeof data.title !== "string") throw new Error("frontmatter missing 'title'");
  if (typeof data.date !== "string") throw new Error("frontmatter missing 'date'");
  return { title: data.title, date: data.date, body: content };

export interface ImportResult { kind: "article" | "nippo"; slug: string; ok: boolean; reason?: string }

interface Importer {
  upsertArchivedArticle(input: { title: string; slug: string; date: string; body: string }): Promise<void>;
  upsertArchivedNippo(input: { title: string; slug: string; date: string; body: string }): Promise<void>;
}

export async function importDir(dir: string, kind: "article" | "nippo", importer: Importer): Promise<ImportResult[]> {
  const results: ImportResult[] = [];
  const files = fs.readdirSync(dir).filter(f => f.endsWith(".md"));
  for (const file of files) {
    const raw = fs.readFileSync(path.join(dir, file), "utf-8");
    try {
      const { title, date, body } = parseFrontmatter(raw);
      const slug = kind === "article" ? file.replace(/\.md$/, "") : date;
      if (kind === "article") {
        await importer.upsertArchivedArticle({ title, slug, date, body });
      } else {
        await importer.upsertArchivedNippo({ title, slug, date, body });
      }
      results.push({ kind, slug, ok: true });
    } catch (e) {
      results.push({ kind, slug: file, ok: false, reason: (e as Error).message });
    }
  }
  return results;
}

const PATTERNS: Array<[RegExp, (m: RegExpMatchArray) => string]> = [
  [/^\/article\/([^/]+)\/?$/, (m) => `/archive/article/${m[1]}`],
  [/^\/nippo\/([^/]+)\/?$/, (m) => `/archive/nippo/${m[1]}`],
];

export function mapOldUrl(pathname: string): string | null {
  for (const [re, build] of PATTERNS) {
    const m = pathname.match(re);
    if (m) return build(m);
  }
  return null;
}
